Section 01

The vision · terminal that knows me

After three weeks of stock Pi, my terminal still feels like running someone else's tool. SumoCode is the persona, memory, and visual identity that makes it mine — and keeps it identical across every machine I touch.

Problem

No persistent identity. Every Pi session starts from zero. The agent doesn't remember TypeScript strict, pnpm-not-npm, London BST, or which company / project I work on.

No visible state. I can't tell at a glance whether the agent is thinking, running a tool, or waiting for approval. Cursor and OpenCode have preattentive status; stock Pi doesn't.

No cross-machine continuity. Mac mini and MacBook need manual setup duplication: extensions, auth, env vars, packages. Memory and preferences live wherever I happen to be.

Solution

Persona layer. Zeus, in the Temple of SumoDeus. Lives in APPEND_SYSTEM.md, applies to every session, identical across machines.

UI surface. Custom footer with state-dot. Right sidebar (or bottom on portrait monitors). Theme-aware working indicator. Three themes for three moods.

Persistent memory. Local Remnic daemon auto-extracts durable facts. Plain markdown. Append-only. Synced via private repo.

One-command bootstrap. git clone && ./bootstrap.sh on a fresh machine; SumoCode lands identical.

Section 05

Memory · append-only by design

Episodes are timestamped immutable files. Entity files use append-only dated sections. The only file that can genuinely conflict is identity.md — and it's intentional that you notice when identity changes.

~/sumocode-config/memory/ ├── identity.mdmanual writes only · cross-project preferences · merge conflicts ARE expected ├── entities/ │ ├── main-app.mdappend-only sections per timestamp · git merges trivially │ ├── readyx.md │ └── stitch.md └── episodes/ ├── 2026-04-25T1432-async-await-pattern.mdextracted by mini · immutable ├── 2026-04-25T1810-promise-all-fetches.mdextracted by macbook · immutable · zero conflict possible └── …

Extraction trigger: Pi session_end event. Model: anthropic/claude-haiku-4-5 (~$0.0005/session). LLM exposure in v1: none — memory is ambient context via the sidebar and system-prompt injection. A memory_search tool is a v0.3+ consideration.

Section 06

Sync · private repo, machine-aware

Personal preferences sync via private GitHub. Physical screen state stays per-machine. Secrets never leave the machine they were issued to.

Synced · sumocode-config repo

  • pi-agent/APPEND_SYSTEM.md
  • pi-agent/settings.json
  • pi-agent/mcp.json (env-templated)
  • pi-agent/extensions/*
  • pi-agent/themes/*
  • pi-agent/prompts/*
  • memory/identity.md
  • memory/entities/*.md
  • memory/episodes/*.md
  • sumocode.json (active theme)

Personal preferences. Memory. Persona. Travels with me.

Machine-local · never synced

  • ~/.pi/agent/auth.json
  • ~/.pi/agent/sessions/
  • ~/.pi/agent/git/ (re-cloned)
  • ~/.pi/agent/mcp-cache*.json
  • ~/.sumocode/local-config.json
  • ~/.zshrc env vars

Secrets. Caches. Sidebar position (depends on physical screen orientation). Per-machine state.

Conflicts are designed away by file structure, not papered over by merge strategy. — Q11 resolution, append-only memory model

Section 10

Out of scope · v2+ only when real frustration drives it

Each item is intentionally deferred. Building before dogfooding is guessing.

Voice input/output Realtime API. Mic permissions. v2 if I miss talking to it.
Proactive behaviors Auto-summarize on git pull. Suggest fix on test fail. Premature without dogfood data.
Time-of-day auto-theme Cathedral by day, Obsidian by night. Build only if I find myself manually switching.
Per-project theme overrides Different repo, different theme. Wait for a real reason.
User-defined custom themes Only the 3 hardcoded themes ship. No external authoring schema.
ACPX-aware persona variants Different default when invoked by OpenClaw. Pi's TTY detection covers the structural case.
Settings UI Hardcoded TS constants edited as code. Settings UIs are an attractive nuisance.
Custom command palette tomsej's leader-key already auto-discovers /sumo:* commands.
