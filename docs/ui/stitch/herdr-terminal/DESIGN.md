# Herdr Terminal — Design Contract (v7)

- **Theme name:** `herdr`
- **Display name:** Herdr Terminal
- **Description:** Electric-green operator terminal — phosphor focus, amber execution, sharp hacker chrome.
- **Source of truth for tokens:** `src/themes/herdr.ts`
- **Bible target:** `docs/ui/bible/theme-herdr-active.html` → `docs/ui/bible/renders/theme-herdr-active.png` (generated by `scripts/gen-bible-theme-herdr.mjs`)
- **Plan:** issue #312 (Plan 073, v7 palette realignment)

## Intent

Herdr Terminal is a focused operator console. After the live Ghostty + Herdr
canary (Mac Mini approval, MacBook parity), the approved direction is:

- green-black chassis and layered green-black surfaces;
- **electric green as the dominant foreground, focus, frame and cursor signal**;
- luminous amber for tools, warnings and durable learning;
- warm red for approval, failure and interruption;
- sharp terminal chrome — no teal, blue, cyan, purple, gradients or rainbow;
- restrained glow only from terminal rendering (no CSS text-shadow, scanline
  animation or Matrix-rain effects);
- semantic hierarchy must survive without relying on colour alone.

This **supersedes** the original Plan 073 cyan/mint/off-white proposal. The live
canary proved that electric-green body text is the preferred product direction.

### Accessibility derivatives

Two SumoCode text colours are deliberate derivatives of the host literals,
which fail the 4.5:1 automated contract on the lifted surface:

- host-muted `#1FA82F` scores 3.94:1 on `#0F3D17`; `#29B938` scores 4.759:1;
- host error `#FF625F` scores 4.21:1 on `#0F3D17`; `#FF706D` scores 4.582:1.

Host literals remain valid for decorative/non-text use only. SumoCode text
uses the derivatives above; the 4.5:1 contract is not weakened.

### Contrast contract

`foreground`, `foregroundDim`, `accent`, and all five state colours must each
stay ≥ 4.5:1 against `background`, `surface`, `surfaceRecess`, and
`surfaceLifted`. Enforced by `src/themes/herdr.test.ts`.

| Colour | Lowest ratio | Worst surface |
|---|---:|---|
| `#39FF14` | 9.110:1 | `#0F3D17` |
| `#29B938` | 4.759:1 | `#0F3D17` |
| `#FFB000` | 6.742:1 | `#0F3D17` |
| `#FF706D` | 4.582:1 | `#0F3D17` |
| `#FFD166` | 8.566:1 | `#0F3D17` |

`divider` is exempt only because it is decorative and never the sole carrier
of text, focus or state.

## State-colour meanings

| State | Colour | Meaning |
|---|---|---|
| idle | green `#29B938` | ready / healthy (quieter than active focus) |
| thinking | electric green `#39FF14` | active reasoning / routing |
| tool | amber `#FFB000` | tool execution / warning |
| approval | red `#FF706D` | interruption / danger / failure |
| learning | bright amber `#FFD166` | durable write / learned state |

Electric green is the dominant active/focus signal. Amber is secondary and
must not replace focus green. Red is reserved for approval/failure.

## Chrome

Single-cell, width-stable characters only (unchanged from v6):

- Frame: `┌ ┐ └ ┘ ─ │` (sharp corners)
- Section sigils: `>` context · `#` memory · `@` mcp · `$` session · `%` registry
- `sectionTracked: false`
- Rule: `─` · Tab active: `▸` · Tab inactive: `·` · Bullet: `>`

## Working indicator

Eight-frame ASCII packet progression at 110 ms, coloured `states.thinking`
(`#39FF14`):

```
. : + * # % @ >
```

All frames unique, width 1. Reduced-motion behaviour follows the shared
indicator contract — no Herdr special-casing outside the bundle.

## Prohibited drift

- **Any cyan, teal, blue or purple** in the Herdr bundle, target or scenario
  gates (this is the specific regression v7 corrects).
- Matrix rain, falling glyphs.
- Glow effects, text shadows, RGB gradients, chromatic aberration.
- Layout changes of any kind — Herdr recolours the locked V2 layout only.
- Cathedral ornament (fleur-de-lis bullets, rounded frames, tracked headers).
- Using amber as the focus signal, or red for anything but approval/failure.
- Double-width or ambiguous-width glyphs in chrome or indicator.
- Reintroducing warm off-white body text — body copy is electric green.
