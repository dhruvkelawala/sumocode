## Synthesis summary

The RPC host at b52ed45 is architecturally coherent — clean client/pump/shell layering, a fail-closed approval gate, and a shared input router — but the seam removal dropped roughly a dozen responsibilities that main's InteractiveMode patch used to carry, and the result is a host that works only for short, happy-path sessions. The most damaging cluster is transcript correctness: agent_end replaces the committed transcript with run-only messages and no session operation (/new, /fork, /clone, /switch) ever rehydrates, so history is wrong after the first completed exchange and stale or mixed after any session switch. The streaming render path compounds this with a full ChatPager teardown, fresh Markdown parses, and a forced scroll-to-bottom on every token, making long sessions both slow and unreadable while the agent works. Approval integrity has two silent-deny holes (palette routing above focus gates, close-on-show overlay slot) that are fail-closed but resolve tool approvals the user never answered. Process robustness is thin: an idle child crash produces a silent zombie with dangling modals, stdin EPIPE and synchronous render throws are uncaught, and a wedged terminal (alt-screen plus raw mode) is the failure mode. Input fidelity has regressed measurably versus main — no pi keybindings (Ctrl+D/Ctrl+V/Ctrl+Z/user overrides), no mouse selection or OSC52 copy, mangled split-codepoint CJK/IME input, and Apple Terminal Shift+Enter submitting instead of newlining. Legacy debt is small and well-fenced: two confirmed-dead items are safe to delete now, while the seam-era controller surface and chat-viewport-controller must survive as the oracle until the incremental-sink and selection batches land. Critically, pending plans 028-031 cover almost none of these findings, and two fixes (stdin encoding, Apple Terminal normalization) sit in code plan 030 explicitly excludes, so they must be landed or annotated before that migration or they will be silently re-lost. The ten batches above are ordered smallest-first and sequenced so correctness (Batch 7) precedes the render rebuild (Batch 9), which precedes the selection port (Batch 10)."
