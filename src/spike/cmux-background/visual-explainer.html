cmux-visible background tasks

Research spike for SumoCode: let the orchestrator spawn long shell work via bg_task, track logs and wakeups in the parent session, and optionally mirror execution into a live cmux split — without pi-subagents overhead.

branch: spike/cmux-background-tasks base: pi-background-tasks v1.5.0 cmux ref: pi-cmux + opencode-cmux

Problem → proposal

What Dhruv wants Orchestrator starts background shell work. Parent keeps chatting. Optional visible cmux split/tab to watch live output.
What we reject pi-subagents for every test run. External tmux orchestration outside Pi tools. Slash-only pi-cmux without model access.

Proposal: fork @vanillagreen/pi-background-tasks, add visible: true spawn path using cmux split + log-tee wrapper (this spike’s PoC).

  O->>B: spawn(visible=true, command)
  B->>C: identify + list-panes
  B->>C: new-split right
  B->>C: respawn-pane (wrapper cmd)
  C->>S: live terminal output
  S->>L: tee stdout/stderr
  B->>L: tail log / read exit.code
  B->>O: wake on completion

opencode-cmux vs our fork

opencode-cmux pattern Listens for session.created with parentID. Opens split, types opencode attach … --session. Works because child is a resumable agent session.
SumoCode shell pattern No attach target. Use pi-cmux’s respawn-pane --command with buildVisibleTaskCommand() (tee + exit marker). Store surfaceRef on task snapshot.

cmux hooks pi install is for session restore — not split creation. Both opencode-cmux and pi-cmux call the cmux CLI directly.
